Git — это распределенная система управления версиями, которая позволяет отслеживать изменения в файлах и координировать работу над ними между несколькими людьми.

1. Настройка и инициализация

- git config --global user.name "Ваше Имя": Устанавливает имя пользователя для всех репозиториев.

- git config --global user.email "ваш@email.com": Устанавливает email пользователя для всех репозиториев.

- git init: Создает новый локальный репозиторий Git в текущей директории.

- git clone url: Клонирует существующий репозиторий с удаленного сервера.

2. Работа с изменениями (Staging Area)

- git status: Показывает текущее состояние файлов: измененные, проиндексированные, неотслеживаемые.

- git add файл: Добавляет указанный файл в область подготовленных изменений (Staging Area).

- git add . : Добавляет все измененные и новые файлы в Staging Area.

- git reset файл: Убирает файл из Staging Area, оставляя изменения в рабочей директории.

- git diff: Показывает непроиндексированные изменения.

- git diff --staged: Показывает проиндексированные изменения (те, что в Staging Area).

3. Коммиты (Commits)

- git commit -m "Сообщение": Создает коммит из проиндексированных изменений с указанным сообщением.

- git commit -a -m "Сообщение": Индексирует и коммитит все отслеживаемые файлы одним шагом.

- git commit --amend: Изменяет последний коммит (полезно для исправления сообщения или добавления пропущенных файлов).

4. Просмотр истории

- git log: Показывает полную историю коммитов.

- git log --oneline: Показывает сжатую историю коммитов (по одной строке).

- git show код_коммита: Показывает изменения, внесенные в конкретном коммите.

- git blame файл: Показывает, кто и когда менял каждую строку в файле.

5. Ветки (Branches)

- git branch: Показывает список всех локальных веток.

- git branch имя_ветки: Создает новую ветку.

- git checkout имя_ветки: Переключается на существующую ветку.

- git checkout -b новая_ветка: Создает новую ветку и сразу переключается на нее.

- git switch -c новая_ветка: Современный аналог git checkout -b.

- git merge ветка: Объединяет указанную ветку с текущей.

- git branch -d ветка: Удаляет локальную ветку (после успешного слияния).

- git branch -D ветка: Принудительно удаляет локальную ветку.

6. Удаленный репозиторий (Remote)

- git remote add origin url: Добавляет удаленный репозиторий с именем origin.

- git push -u origin ветка: Отправляет коммиты из локальной ветки на удаленный сервер (и связывает их).

- git pull: Получает изменения с удаленного репозитория и автоматически объединяет их с текущей локальной веткой.

- git fetch: Получает изменения с удаленного репозитория, но не объединяет их.

- git push origin :ветка: Удаляет ветку на удаленном репозитории.

7. Отмена изменений (Undoing Changes)

- git reset --soft код_коммита: Отменяет коммиты, но сохраняет изменения в Staging Area.

- git reset --mixed код_коммита: Отменяет коммиты и изменения в Staging Area (по умолчанию).

- git reset --hard код_коммита: Отменяет коммиты, Staging Area и все изменения в рабочей директории.

- git revert код_коммита: Создает новый коммит, который отменяет изменения, внесенные указанным коммитом.

- git restore файл: Отменяет изменения в рабочем каталоге для указанного файла (возвращает к состоянию последнего коммита или Staging Area).

- git clean -f: Удаляет неотслеживаемые файлы из рабочей директории.
