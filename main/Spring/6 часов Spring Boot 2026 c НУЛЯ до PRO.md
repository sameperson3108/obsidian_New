Inversion of Control (IoC) - принцип, согласно которому управление объектами и их зависимостями передаётся приложению, а не осуществляется в ручную в коде. Реализуется в Spring за счёт ApplicationContext.

ApplicationContext - контейнер, который управляет жизненным циклом объектов программы (бины), внедрением зависимостей и обработкой аннотаций и конфигураций

Способы создания бинов: 
- при помощи аннотаций (@Component, @Service, @Repository, @RestController)
- xml configuration (устаревший способ)
- специальный конфигурационный класс @Configuration

Dependency Injection - принцип, который позволяет внедрять зависимости объектов извне, а не создавать их внутри самого объекта.
Это помогает уменьшить связанность компонентов, улучшить тестируемость и модульность кода.

Способы Dependency Injection:
- @Autowired - Основная аннотация для указания внедрения зависимостей. 1. Fielde Injection - ставить над полем. 2.Setter Injection. 3 Constructor Injection

Из браузера можно получать только get запросы


http - протокол передачи данных в сети интернет. Протокол - четкий набор правил, что есть в запросе и что есть в ответе. Клиент - тот, кто посылает запрос. Сервер - тот, кто отправляет ответ

Основные типы запросов:

- DELETE - удаление

REST(Representational State Transfer) - архитектурный стиль, использующий HTTP для взаимодействия с ресурсами

Принципы REST:
- Клиент-Серверное взаимодействие
- Statless сервер - сервер не запоминает предыдущие запросы
- Кэширование запросов
- Единый интерфейс - всё является ресурсами. У каждого ресурса свой путь
- Многослойная система

JSON (JavaScript Object Notation) - легковесный человеко-читаемый формат для обмена данными. Используется для передачи данных между клиентом и сервером в веб-приложениях, особенно при REST взаимодействии.

Преимущества:
- Легковесность
- Удобочитаемость
- Широкая поддержка
- Гибкость и расширяемость
- Совместимость с веб-разработкой



JPA (java persistence api) - спецификация для управления реляционными данными в java-приложениях. jpa представляет собой набор api и аннотаций для маппинга объектов java в реляционные базы данных (ORM(Object Relational Mapping))

ORM фреймворк позволяет работать с базой данных, используя объектно-ориентированный подход, снижает количество ошибок с ручным написанием sql запросов

Hibernate - популярная реализация JPA, предлагающая дополнительные возможности и оптимизации. Hibernate предоставляет богатый функционал для работы с базами данных, включая автоматическое создание схемы, кэширование, поддержку полиморфизма и наследования

Docker - платформа для разработки, доставки и эксплуатации приложений с использованием контейнеризации. Позволяет разработчикам упаковывать приложения и все их зависимости в контейнеры, которые можно запускать в любой среде, обеспечивая их стабильную работу вне зависимости от конфигурации систем.

Преимущества Docker:
- Портативность: Образы могут быть запущены на любой системе с установленным Docker
- Изоляция: Контейнеры изолируют приложение и его зависимости от системы и других контейнеров
- Легкость и быстрота: Контейнеры запускаются значительно быстрее виртуальных машин, так как они не требуют полной операционной системы

Dockerfile - текстовый файл, содержащий набор инструкций для сборки Docker-образа

При помощи команды Docker build можно собрать Dockerfile в Docker image

Docker image - шаблон, который используется для создания контейнеров. Образ содержит все, что необходимо для запуска приложения, включая код, runtime, библиотеки и зависимости, а также параметры конфигурации.

При помощи команды Docker run можно запустить Docker image в Docker Container

Контейнер Docker - экземпляр, созданный на основе Docker образа.


Spring boot data jpa - старте из spring boot, который упрощает использование JPA в приложениях spring. Этот стартер предоставляет необходимые зависимости и автоматическую конфигурацию, чтобы интегрировать Spring data jpa в приложение

Предоставляет функции:
- Автоматическая конфигурация подключения к БД
- Использует под капотом JPA, hibernate в качестве реализации
- Предоставляет интерфейсы репозиториев, которые упрощают создание функций CRUD
- Управление транзакциями
- Поддержка сортировок и постраничной выборки
- Возможность писать свои более сложные запросы

Слои приложения - абстракция, для разделения приложения, разделения логики между ними
Основных слоя 3: 	
- API(Controller, принимает GET PUT POST DELETE, принимает http. Его задача принимать http запросы и корректно на них отвечать)
- Service Layer(Бизнес логика)
- Database Layer

Аннотация @Querry в Spring Datat JPA представляет мощный способ определения запросов к базе данных непосредственно в интерфейсе репозитория
Предоставляет функционал:
- Поддержка JPQL и SQL
- Использование именованных параметров
- Возможность писать модифицирующие запросы
